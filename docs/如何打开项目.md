# StoryForge 故事熔炉 · 如何打开项目

> 三种方式，从最简单到最完整。大多数人用**方式一**就够了。

---

## 方式一：直接用在线版（最简单，不用下载）

打开浏览器访问：

**https://yuanbw.vercel.app/storyforge/**

打开即用，不用安装任何东西。所有数据存在你自己的浏览器里（本地），不上传服务器。

> 注意：换浏览器、换电脑、清浏览器数据，数据不会跟着走（因为存在本地）。重要内容记得用工具内的「导出」功能存成文件。

---

## 方式二：本地运行（自己电脑上跑，永久有效）

适合想离线用、或想改代码的人。一共 5 步，**第一次配置好之后，以后每次只要做第 4-5 步**。

### 第 1 步：安装 Node.js（只需一次）

1. 打开 https://nodejs.org/zh-cn
2. 下载 **LTS（长期支持）** 版本
3. 双击安装，一路下一步即可（Windows 安装时确保勾选 "Add to PATH"）
4. 验证：打开终端（见下方），输入 `node -v`，能显示版本号（如 `v20.x.x`）就成功了

> 终端怎么打开：
> - **Windows**：在项目文件夹里，按住 `Shift` + 右键 → 「在此处打开 PowerShell / 终端」
> - **Mac**：在「访达」里进入项目文件夹 → 右键 →「服务」→「在文件夹中打开终端」

### 第 2 步：下载项目

- 打开 https://github.com/yuanbw2025/storyforge
- 点绿色的 **「Code」** 按钮 → **「Download ZIP」**
- 下载后**解压**到一个你记得住的文件夹

> 想要日后一键更新代码的，可以用 `git clone https://github.com/yuanbw2025/storyforge.git`（需先装 git，非必须）。

### 第 3 步：安装依赖（只需一次）

在项目文件夹里打开终端，输入：

```bash
npm install
```

等它跑完（约 1-2 分钟，需要联网）。

> **国内网络慢/装不上**，先换国内镜像再装：
> ```bash
> npm config set registry https://registry.npmmirror.com
> npm install
> ```

### 第 4 步：启动

```bash
npm run dev
```

### 第 5 步：打开浏览器

启动后访问：

**http://localhost:5175/storyforge/**

（如果浏览器没自动打开，手动复制这个地址到浏览器即可。）

> 关闭终端窗口 = 停止运行。下次再用，只要重复**第 4-5 步**（不用再装 Node 和依赖）。

---

## 方式三：用 Trae / VS Code 等编辑器（适合会用 IDE 的人）

1. 用 Trae 或 VS Code 「打开文件夹」选中解压后的项目
2. 打开编辑器自带的终端
3. 同方式二的第 3-5 步：`npm install` → `npm run dev` → 开浏览器

> Trae 能让 AI 帮你执行命令，对不熟悉命令行的人友好；但本质和方式二一样，都是 `npm install` + `npm run dev`。

## 常见问题

**Q：之前的 start.bat 双击没反应/报错？**
A：start.bat 只能 Windows 用，且依赖系统环境，容易因为没装 Node、被杀毒软件拦截、或网络问题失败。**推荐改用方式二的手动命令**——更稳定、Windows/Mac 通用、永久有效。

**Q：`npm` 不是内部或外部命令 / command not found？**
A：说明 Node.js 没装好或没加入 PATH。重装 Node.js（Windows 勾选 Add to PATH），重开终端再试。

**Q：端口 5175 被占用？**
A：关掉其他占用该端口的程序；或它会自动换一个端口，看终端提示的地址。

**Q：我的小说数据存在哪？会丢吗？**
A：存在你浏览器的本地数据库（IndexedDB），不上传服务器。**只要不清浏览器数据就一直在**。但换电脑/换浏览器不会同步，重要内容请用工具内「导出」存成文件备份。

**Q：在线版和本地版有区别吗？**
A：功能一样。在线版免安装；本地版可离线、可改代码。两者数据**各自独立**（在线版存在线浏览器，本地版存本地浏览器）。
