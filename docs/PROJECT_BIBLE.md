## 🎯 一句话定义

**赛博飞剑是一款通过摄像头手势操控飞剑鞭子的 2.5D 横版动作游戏。**
玩家用右手挥舞飞剑鞭子，鞭子既是武器也是移动工具——鞭子带着角色飞行，鞭尾抽打敌人造成伤害。

---

## 🚫 绝对不可违反的设计原则 (Red Lines)

| # | 原则 | 解释 |
|---|------|------|
| 1 | **玩家只操控飞剑鞭子，不操控角色** | 角色是鞭子的附属物理对象，不是操控主体。永远不要给角色独立的地面移动能力。 |
| 2 | **手势不是键盘替代品** | 每个手势都必须有独特的物理意义。如果一个操作用键盘能做得更好，就不应该存在。 |
| 3 | **操控成本来自物理，不来自数值** | "抽飞自己"的成本是失去位置控制权，不是扣一个"移动力"数字。 |
| 4 | **握拳=放弃一切换取防御** | 护盾激活时玩家无法移动和攻击，这是核心风险/收益权衡。 |
| 5 | **永远只有一个操控对象** | 不存在"切换操控主体"的情况。手指→鞭子→影响世界，始终如此。 |
| 6 | **单手操控** | 右手，手肘撑桌，30-40cm 距离摄像头。不要设计需要双手的操作。 |

## 🎮 核心 Game Loop

```
挥鞭移动（手指移动→鞭子→角色跟随）
    ↓
发现敌人 → 挥鞭攻击（鞭尾碰敌人=伤害）
    ↓
需要位移 → 甩鞭抽角色（角色弹射飞行）
    ├── 速度够快 → 气流罩 → 撞敌=伤敌+自己少受伤
    └── 速度不够 → 无气流罩 → 碰敌=受伤
    ↓
敌人攻击 → 选择：
    ├── 继续挥鞭闪避（保持攻击能力，但可能被打中）
    ├── 握拳格挡（牺牲移动和攻击，消耗护盾）
    └── 精防（在攻击前~100ms握拳=不耗盾+大量怒气）
    ↓
怒气满 → 一指定江山（拇指+食指90°）→ 全屏飞剑齐射
    ↓
循环
```

## 🕹️ 操控方案

### 输入设备
- 普通 USB/笔记本摄像头
- MediaPipe Hands（单手，`maxNumHands: 1`）
- 右手，手肘撑桌面，摄像头距离 30-40cm

### 三种核心手势

| 手势 | 检测方式 | 游戏动作 |
|------|---------|---------|
| **自然手型**（默认） | 非握拳 且 非一指定江山 | 挥鞭操控（移动+攻击） |
| **握拳** ✊ | 所有手指弯曲 | 护盾/格挡（牺牲移动和攻击） |
| **一指定江山** 👆 | 拇指+食指伸展呈90°，其余三指弯曲贴掌 | 怒气大招（全屏飞剑齐射） |

> 📄 详细手势图解见 `docs/gestures/all_gestures_reference.html`

### 操控映射
- **手指位置**（食指尖 landmark 8）→ 鞭子握柄端位置
- **手指运动**产生的 `fingerHistory[N]` lerp 链 → 鞭子的物理绳体
- **鞭尾速度** → 攻击伤害（速度越快伤害越高，从尾部向握柄衰减）
- **鞭子经过角色** → 角色被带动/弹射

## 📊 资源系统

| 资源 | 获取方式 | 消耗方式 |
|------|---------|---------|
| **血量 (HP)** | 恢复点/击杀掉落 | 被敌人攻击 |
| **护盾 (Shield)** | 待定（自然回复/击杀回复） | 非精防格挡时消耗 |
| **怒气 (Rage)** | 攻击敌人(少) + 被攻击(中) + 精防(多) | 释放大招 |

精防机制：在敌人攻击命中前 ~100ms 内握拳 → 不消耗护盾值 + 大量怒气

## ⚔️ 飞剑形态

| 形态 | 外观 | 状态 |
|------|------|------|
| **佩剑** | 一把剑背在角色背上 | 非战斗默认 |
| **涡流** | 飞剑散开旋转 | 过渡形态 |
| **鞭子** | 涡流收束为剑链鞭 | 战斗中（核心形态） |

## 🌍 场景结构

- **横版卷轴**探索关卡（有平台、地形、分支路径）
- **Boss 关卡**（封闭场地，有边界/结界）
- 角色可利用弹射穿越距离、到达高处平台
- 弹射后画面边缘反弹（Boss 战中反弹墙=结界壁）
