## 🛑 第一戒律：强制行动前置沟通法则 (Communicate BEFORE Acting)

由于你（基础级 AI）的推理窗口存在限制，极容易擅自做主并改坏原有的完美系统。为了防止项目崩溃，你被强加了最高优先级的**三大前置铁律**。

在接到用户的任何开发要求时，**你必须在执行任何修改前，停下来向用户发送一段清晰的“计划通报”。** 通报必须严格包含以下内容：
1. **你打算做哪些具体的操作**：修改哪个变量？重写哪个组件？调用哪个函数？
2. **全局影响面核对承诺**：你必须主动写出“我已核对当前行动与项目原有的架构及构建逻辑（如 `build.mjs` 和 `telemetry-sync` 代理）**没有任何冲突**。如果有潜在的架构断裂风险，你必须马上指出并警告用户！
3. **请求放行**：必须等待人类开发者回复了“同意”或类似指令后，你才能调用生成和写入代码的权限。

*※ 注：如果人类开发者说“你去规划一下”，你只需在你的文本内容中写明计划，然后停下。*

## 🏛 项目现状与底层生命周期 (You MUST understand this first)

在动手前，你必须透彻理解这个巨无霸生态系统的构造：

### 1. 这是一个 Monobuild (主单体构建) 体系
它绝不仅仅是一个简单的单页静态网站，它是**“主网站 + 子系统工具集”**的聚合体。
项目的宿主在 Vercel（核心项目名通常是 `my-website`），其部署靠的是根目录自己手写的 `build.mjs`。
- **构建工作流**：Vercel 跑 `node build.mjs` 时，脚本会**层层潜入**子文件夹（如 `infiniteskill/`），使用 `npm run build` 进行编译，最后再把编译产物复制并聚合到统一的 `public/` 目录下。
- **冲突防范要求**：不要瞎改根目录的内容想把它变成 Vite 项目，它不是。不要为了修改静态网页而去触碰 `build.mjs` 内的核心编译流。

### 2. 子项目 InfiniteSkill 工具架构
这是挂载在主站下的关键应用，路径位于根目录的 `infiniteskill/` 文件夹下。
- **技术栈**：严格采用 `Vite 6` + `React 19` + `TypeScript` + `TailwindCSS v4`。
- **部署表现**：在 Vercel 上通过 `vercel.json` 将访问 `/infiniteskill/(.*)` 的用户正确导向给这个 React SPA，确保客户端路由不挂。

### 3. API 安全与鉴权设计 (生命线：绝不能泄露明文)
为了彻底隐藏敏感的系统级 LLM API Keys，本项目独创了一套 Vercel 后端中转机制。该机制经历过数次因为硬编码导致的 Google 官方封号血泪史。
**所有后来的 AI 必须把以下规范刻进代码本能中：**
- **只能引用环境变量**：在任何文件（尤其是后端的 Serverless 如 `/api/v2/telemetry-sync.js`）里调用 Gemini API 时，**绝对、无论如何不允许写入 `const API_KEY = "AIza..."` 这样的保底硬编码字符串**。所有的通信密钥只允许且强制通过 `process.env.SECRET_GEMINI_KEY` 获取。哪怕没有取到，你也只能通过抛出 500 并在日志中要求用户配置 Vercel 环境变量来阻断运行，严禁添加任何回退字符串。
- **纯粹的 Native 接口**：后台转发已抛弃错误百出的 OpenAI Adapter（兼容接口），全面换成了 Gemini 原生调用 `https://generativelanguage.googleapis.com/v1beta/models/[model]:generateContent`。不要把它擅作主张重构成 OpenAI SDK 形式！
- **响应格式伪装**：这是由于前端项目 (`infiniteskill/src/lib/skill-compiler.ts`) 为了兼容历史 OpenAI 依赖，期待返回格式为 `{ choices: [{ message: { content: "..." } }] }`。你的任务是维护这个后端的原样：拦截发给 Gemini，拿到结果后再伪装成前面的格式送给前端。

## 📋 开发者代码手势规范 (Coding Style Rules)

当你终于获得了人类的修改许可，进入写代码的步骤时：

1. **绝对详细与保守**：在提供代码补丁时，你必须尽量保持原有的代码整洁性。如果要求重构，不要遗漏现有的正常防呆容错语句。
2. **杜绝占位符**：如果有不清楚的业务逻辑，立刻返回去问用户。如果你为了省事写了诸如 `/* 后续逻辑省略 */` 导致整段代码崩溃，将被视作重大事故并被清退重启。
3. **工作记忆台账强同步制**：一旦你完成了一个工具调用（代码替换或生成）并生效了，在此之前你必须强制更新 `WORKING_MEMORY.md` 顶部的【开发日志追踪台账】板块。格式必须是精确到时分秒：`[15:20:00] 进行了什么改动，目前项目推进到了哪一步。`
